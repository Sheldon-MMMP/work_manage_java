# OKR个人生活管理APP实现计划

## 技术栈与依赖

* **后端框架**: Spring Boot 4.0.1

* **数据库**: PostgreSQL

* **ORM**: Spring Data JPA

* **Web层**: Spring Web

* **安全认证**: Spring Security + JWT

* **时间处理**: Java 8 Time API

* **构建工具**: Maven

## 数据库设计

### 1. 用户表 (user)

* id (主键)

* username (用户名)

* password (密码)

* email (邮箱)

* created\_at (创建时间)

### 2. 目标表 (objective)

* id (主键)

* user\_id (外键，关联用户)

* title (目标标题)

* description (目标描述)

* status (状态：active/archived)

* created\_at (创建时间)

* updated\_at (更新时间)

* award(奖品， 允许为空)

### 3. 关键结果表 (key\_result)

* id (主键)

* objective\_id (外键，关联目标)

* title (KR标题)

* description (KR描述)

* weight (权重)

* created\_at (创建时间)

* updated\_at (更新时间)

### 4. 任务表 (task)

* id (主键)

* kr\_id (外键，关联KR)

* title (任务标题)

* description (任务描述)

* status (状态：todo/in\_progress/done)

* start\_time(开始时间)

* end\_time(结束时间)

* created\_at (创建时间)

* updated\_at (更新时间)

### 5. 时间记录 (time\_record)

* id (主键)

* task\_id (外键，关联任务)

* user\_id (外键，关联用户)

* start\_time (开始时间)

* end\_time (结束时间)

* created\_at (创建时间)

### 6. 纪念日表 (anniversary)

* id (主键)

* cover\_url (封面链接)

* user\_id (外键，关联用户)

* title (纪念日标题)

* description (纪念日描述)

* anniversary\_date (纪念日日期)

* created\_at (创建时间)

## API设计

### 1. 认证API

* POST /api/auth/register - 注册

* POST /api/auth/login - 登录

### 2. 目标API

* GET /api/objectives - 获取目标列表

* POST /api/objectives - 创建目标

* PUT /api/objectives/{id} - 更新目标

* DELETE /api/objectives/{id} - 删除目标

* PUT /api/objectives/{id}/archive - 归档目标

### 3. KR API

* GET /api/objectives/{objectiveId}/key-results - 获取KR列表

* POST /api/objectives/{objectiveId}/key-results - 创建KR

* PUT /api/key-results/{id} - 更新KR

* DELETE /api/key-results/{id} - 删除KR

### 4. 任务API

* GET /api/key-results/{krId}/tasks - 获取任务列表

* POST /api/key-results/{krId}/tasks - 创建任务

* PUT /api/tasks/{id} - 更新任务

* DELETE /api/tasks/{id} - 删除任务

### 5. 时间记录API

* POST /api/time-records/record - 记录定时(前端将计时器的结果传递回来，直接将结果写入数据库)

* GET /api/time-records/today - 获取最近时间记录

* GET /api/time-records/today/summary - 获取今日总结

### 6. 纪念日API

* GET /api/anniversaries - 获取纪念日列表

* POST /api/anniversaries - 创建纪念日

* PUT /api/anniversaries/{id} - 更新纪念日

* DELETE /api/anniversaries/{id} - 删除纪念日
